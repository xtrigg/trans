# 上下文回复与英文语音播放实施记录

日期：2026-05-14

## 目标

增强 `我要回复` 区：

- 生成英文回复时可使用左侧会议上下文。
- 增加英文 TTS 播放按钮，让用户可播放 AI 生成英文语音。
- 保持原入口 `https://xcu.ai/trans/`。

## 已实现

### 会议上下文

新增 `使用会议上下文` 开关，默认开启。

生成英文回复时，前端会把左侧最近内容一起发送到后端：

- `contextSource`：左侧原文识别最近文本
- `contextTranslation`：左侧中文翻译最近文本

后端 prompt 会使用上下文保留姓名、时间、地点以及对方刚说过的内容。

### 英文语音播放

新增按钮：

- `播放英文语音`
- `停止播放`

新增播放器：

- `replyAudio`

接口：

- `/api/openai/reply-speech`
- `/api-proxy/api/openai/reply-speech`

Cloudflare Pages Function 使用 OpenAI Speech API：

- Endpoint：`https://api.openai.com/v1/audio/speech`
- 默认模型：`gpt-4o-mini-tts`
- 默认音色：`coral`
- 返回格式：`mp3`

页面明确提示播放的是 AI 生成语音。如果要让 Zoom 听到，需要确认扬声器/麦克风或虚拟音频路由不会产生回声。

### Cloudflare Worker Router

`xcu-trans-router` 新增路由：

- `xcu.ai/api-proxy/api/openai/reply-speech`

### 本地 api-proxy fallback

本地 Express `api-proxy` 同步新增：

- `/api/openai/reply-speech`
- `/api-proxy/api/openai/reply-speech`

线上样例：

中文回复：

```text
我想确认一下明天早上孩子的接送时间。
```

会议上下文：

```text
The pickup time is at 8:30 tomorrow morning.
明天早上八点半接送。
```

生成英文：

```text
I just want to confirm the pickup time for my child tomorrow morning—it's at 8:30, right?
```

短句版：

```text
Just confirming: pickup is at 8:30 tomorrow morning, right?
```

## 当前限制

- 语音播放只在浏览器中播放，不会直接接入 Zoom 麦克风。
- 如果要让 Zoom 远端听到，需要外放到麦克风或后续接虚拟音频设备。
- 纯网页仍不能直接控制 Zoom 静音。
