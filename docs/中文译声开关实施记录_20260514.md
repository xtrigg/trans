# 中文译声开关实施记录

日期：2026-05-14

## 目标

在实时翻译页面增加一个开关，控制是否在本机播放翻译后的中文语音。关闭后页面继续显示原文识别和中文翻译文本，但不再从浏览器播放器听到中文译声。

## 实现

- 在“我听别人”区域增加 `听中文译声` 复选框，默认开启，保持原有体验。
- 启动实时翻译时读取开关状态，并传入前端 `OpenAIRealtimeTranslator`。
- 在运行中切换开关时调用 `setOutputAudioEnabled()`：
  - 开启：解除远端音频静音，并尝试继续播放。
  - 关闭：将远端音频静音并暂停播放器。
- 不向 OpenAI realtime translation session 发送未文档化参数，避免 session 创建失败。

## 注意

根据 OpenAI 当前公开说明，`gpt-realtime-translate` 会返回译音轨和字幕事件，官方文档没有给出稳定的“只返回文本、不生成音频”的 session 配置。因此本次实现重点保证本机不播放中文 TTS。若后续 OpenAI 提供正式 text-only translation 参数，再升级为真正减少输出音频生成。

## 验证

- 新增前端静态测试，确认页面包含 `听中文译声` 开关，并在启动和切换时调用对应逻辑。
- 新增模块测试，确认关闭输出音频时会设置 `remoteAudio.muted = true` 并暂停播放器。
