# 双向会议翻译音频路由方案

日期：2026-05-14

## 核心判断

实时翻译模型现在已经可用，真正的难点是音频路由。

会议里至少有两条完全不同的链路：

1. 别人说话 -> 翻译成中文 -> 送到我的耳朵
2. 我说中文 -> 翻译成英文 -> 送进 Zoom/Teams/Google Meet 的麦克风输入

这两条链路必须隔离。不能让 Zoom 的英文声音再次进入“我说中文”的翻译链路，也不能让翻译出来的英文再被麦克风收进去二次翻译，否则会产生回声、循环、串音和互相干扰。

## 当前网页能做什么

当前 `https://xcu.ai/trans/` 是浏览器麦克风实时翻译：

- 输入：浏览器麦克风
- 输出：浏览器播放器里的翻译声音
- 适合：现场听别人说话，然后翻译到耳机
- 不足：它还不能把“我说中文后生成的英文 TTS”直接送进 Zoom 的麦克风设备

## 今天会议的最稳方案

如果会议就在几个小时后，建议不要临时折腾复杂虚拟声卡链路。最稳的用法是：

1. 用耳机听 Zoom/现场声音，避免外放。
2. 用 `xcu.ai/trans/` 做“听别人说话 -> 中文到耳朵”的辅助。
3. 你要回复时，尽量用短句英文直接说；复杂内容用文字翻译后发 Zoom chat 或会后邮件。
4. 如果必须让对方听英文 TTS，不要让 Zoom 同时使用你的真实麦克风；否则对方会听到中文原声和英文译声混在一起。

这是保守方案，但最不容易在正式会议里翻车。

## 双向实时方案需要的标准音频路由

理想链路如下：

### A. 听别人

Zoom 远端声音 -> 系统音频捕获/虚拟线 -> 翻译引擎 -> 中文 TTS -> 我的耳机

### B. 我说话

我的麦克风 -> 翻译引擎 -> 英文 TTS -> 虚拟麦克风 -> Zoom 麦克风输入

### 必须隔离

- Zoom 的 Speaker 必须去耳机或独立监听通道。
- Zoom 的 Microphone 不能选真实麦克风，应该选“虚拟麦克风”。
- 真实麦克风只给翻译器使用。
- 翻译器输出给 Zoom 的英文 TTS 不能再被翻译器麦克风录进去。
- 必须戴耳机，不能开扬声器。

## Windows 上的可执行路线

### 路线 1：VB-CABLE / Voicemeeter + 浏览器或桌面助手

需要安装虚拟声卡：

- VB-CABLE：提供一条虚拟音频线，播放到 Cable Input 的声音会出现在 Cable Output。
- Voicemeeter：更适合多路混音、监听、把一路音频送耳机、另一路送虚拟麦克风。

Zoom 设置：

- Speaker：耳机，或 Voicemeeter 的监听输出
- Microphone：虚拟麦克风，例如 VB-CABLE Output 或 Voicemeeter Output

翻译器设置：

- 输入 1：Zoom 远端声音，用于翻译给我听
- 输入 2：我的真实麦克风，用于翻译给别人听
- 输出 1：中文 TTS 到我的耳机
- 输出 2：英文 TTS 到虚拟麦克风，让 Zoom 发送给别人

当前网页还没有这些设备选择和双通道能力，需要继续开发。

### 路线 2：做一个 Electron/Tauri 桌面助手

这是产品级路线。桌面助手可以更稳定地处理：

- 系统音频捕获
- 麦克风捕获
- 多路音频输出
- 虚拟设备选择
- 静音/按键说话/防回声
- Zoom/Teams/Meet 兼容

网页适合快速验证模型效果；真正要做会议双向翻译，桌面助手更合适。

### 路线 3：Zoom 原生翻译能力

Zoom 自己有语言口译/翻译相关能力，但通常依赖主持人、账号能力、会议设置或 Zoom 自身功能可用性。它适合正式多人会议，但不一定能满足“我个人把中文自动翻成英文发出去”的私人工具需求。

## 推荐产品改造

下一阶段应该把工具拆成两个模式：

### 模式 1：我听别人

- 选择输入：麦克风 / 系统音频 / Zoom 音频
- 选择输出：耳机
- 目标语言：中文
- UI 显示：原文、中文、延迟

### 模式 2：我说给别人

- 选择输入：我的麦克风
- 选择输出：虚拟麦克风
- 目标语言：英文
- 控制方式：按住说话 / 点击开始说话 / 自动语音检测
- 保护：我不说话时不向 Zoom 输出声音

### 模式 3：全双工

- 同时运行“我听别人”和“我说给别人”
- 必须强制要求耳机
- 必须检测输入输出设备是否相同
- 如果发现回路风险，直接禁止启动

## 明确回答

你说中文，对方听到英文，技术上不是直接“把 TTS 送给 Zoom 软件”这么简单，而是要把英文 TTS 伪装成一个系统麦克风输入。Zoom 只认系统里的麦克风设备，所以需要虚拟麦克风或桌面音频路由。

排除互相干扰的关键是：

- 真实麦克风只进翻译器
- Zoom 麦克风只选翻译后的虚拟麦克风
- Zoom 声音只进你的耳机和“别人说话翻译链路”
- 翻译输出不能外放，必须戴耳机
- 双向链路要分开运行，不能混成一个麦克风输入
