# 静态化部署方案

日期：2026-05-14

## 结论

当前实时翻译页面的前端基本是静态 HTML、CSS、JavaScript，可以部署到 Cloudflare Pages 或 GitHub Pages。

但整个工具不能只靠纯静态站点完成，因为浏览器端不能保存或暴露 `OPENAI_API_KEY`。实时翻译启动前需要一个安全后端接口创建 OpenAI Realtime 的临时 client secret，然后浏览器再用这个临时凭证连接 OpenAI。这个接口现在由 `api-proxy` 提供。

## 当前依赖拆分

| 部分 | 是否静态 | 说明 |
|---|---:|---|
| `public/realtime-translation-poc.html` | 是 | 页面、按钮、状态、播放器 |
| `public/realtime-translation.mjs` | 基本是 | 浏览器麦克风、WebRTC、调用 session 接口 |
| `/api-proxy/api/openai/realtime-translation/session` | 否 | 必须在服务端读取 `OPENAI_API_KEY` 并向 OpenAI 创建临时 client secret |
| nginx / Docker / Cloudflare Tunnel | 可替代 | 目前用于本机公网访问，但不是必须架构 |

## 方案 A：Cloudflare Pages + Pages Function/Worker（推荐）

把 `public/` 作为 Cloudflare Pages 静态站点部署；把现在 `api-proxy` 里创建 OpenAI realtime session 的逻辑迁移成 Cloudflare Pages Function 或 Worker。

优点：
- 不依赖本机 Windows、Docker Desktop、nginx、Cloudflare Tunnel。
- `xcu.ai/trans/` 可以稳定由 Cloudflare 托管。
- OpenAI key 存在 Cloudflare 变量/Secrets 里，不暴露给浏览器。
- Pages Functions 官方支持在 Cloudflare 网络上运行服务端逻辑；环境变量可通过 Workers & Pages 设置。

缺点：
- 需要把 Express 风格接口改成 Worker/Pages Function 风格。
- 需要处理路由，比如 `/api/openai/realtime-translation/session`。

适合目标：
- 你马上要在手机、家长会、客户沟通场景稳定测试。
- 不想再受本机 Docker Desktop 是否启动影响。

## 方案 B：GitHub Pages + Cloudflare Worker

把静态页面放到 GitHub Pages；把 session 创建接口放到 Cloudflare Worker。

优点：
- GitHub Pages 很适合纯静态 HTML/JS/CSS。
- 代码推 GitHub 后可自动发布。

缺点：
- GitHub Pages 不能安全保存 OpenAI 主 API key，也不能运行这个后端接口。
- 仍然需要 Cloudflare Worker 或其他 serverless 后端。
- 跨域和自定义域名配置会比 Cloudflare Pages 一体化方案多一点。

适合目标：
- 你想把 GitHub 作为主发布入口。
- 后端接口仍愿意单独托管在 Cloudflare Worker。

## 方案 C：继续本机 nginx + Cloudflare Tunnel

保持当前模式：本机 Docker 跑 `nginx`、`api-proxy`、`cloudflared`。

优点：
- 已经部署完成，修改代码马上可见。
- 本地调试方便。

缺点：
- Windows 或 Docker Desktop 一停，公网入口就断，Cloudflare 会报 1033。
- 不适合作为稳定会议工具。

## 推荐路线

建议采用方案 A：Cloudflare Pages + Pages Function/Worker。

第一步只迁移当前 POC：
1. Cloudflare Pages 托管 `public/`。
2. 新增一个 Pages Function/Worker 提供 session 接口。
3. 在 Cloudflare 设置 `OPENAI_API_KEY` secret。
4. 绑定 `xcu.ai/trans/` 或单独子域名，例如 `trans.xcu.ai`。
5. 保留当前本机 Docker 版本作为开发环境和回退方案。

这样迁移后，网页入口不再依赖本机 Docker Desktop，1033 这类 Tunnel connector 离线问题会消失。

## 官方依据

- Cloudflare Pages Functions 可在 Pages 上加入服务端逻辑，用于不跑独立服务器的动态功能。
- Cloudflare Pages 支持通过 Workers & Pages 配置环境变量/Secrets。
- GitHub Pages 适合静态站点发布。
- OpenAI Realtime 浏览器连接应使用短期 client secret，避免把主 API key 暴露给前端。
