# 原入口访问故障排查

日期：2026-05-18

## 结论

原入口应继续使用：

```text
https://xcu.ai/trans/
```

Cloudflare 公网入口本身正常，强制走 Cloudflare 边缘 IP 时，`https://xcu.ai/trans/` 可以返回新版实时翻译页面，并包含：

- 默认实时译声模式
- 话筒选择
- 省钱文字模式备用

## 本机异常原因

当前 Windows hosts 文件中存在 `xcu.ai` 覆盖：

```text
127.0.0.1    xcu.ai
192.168.1.111    xcu.ai
192.168.2.180      xcu.ai
```

浏览器会优先使用 hosts，因此本机打开 `https://xcu.ai/trans/` 时可能命中本机或内网旧 nginx，而不是 Cloudflare Pages/Worker。

## 需要处理

用管理员权限打开：

```text
C:\Windows\System32\drivers\etc\hosts
```

把这三行注释或删除：

```text
# 127.0.0.1    xcu.ai
# 192.168.1.111    xcu.ai
# 192.168.2.180      xcu.ai
```

然后执行：

```powershell
ipconfig /flushdns
```

再打开：

```text
https://xcu.ai/trans/
```

## 说明

Cloudflare Pages 的预览地址只用于部署验证，不应作为日常入口。日常入口统一使用 `https://xcu.ai/trans/`。
