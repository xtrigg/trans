# 省钱文字模式默认改造实施计划

日期：2026-05-15

## 目标

将 `https://xcu.ai/trans/` 的默认会议翻译体验改为省钱文字模式，并改善真实会议阅读体验：

- 默认不生成实时译声音频，避免默认进入高成本实时译声。
- 中文翻译作为主阅读区，占据大部分页面。
- 英文原文默认折叠，仅在用户需要时显示。
- 控制项折叠到右侧抽屉，减少对阅读区域的干扰。
- 增加单次会话 60 分钟上限、倒计时提醒、到点自动停止。
- 增加键盘/PTT 式开始停止切换，方便蓝牙遥控器或映射键使用。

## 架构

保留两条模式：

1. `省钱文字模式`：
   - 优先参考/迁移原始 `public/trans.html` 中已实现的 Whisper 分段转写、Deepgram 实时转写、BLE 麦克风 PCM/WAV 处理和转写合并逻辑。
   - 当前第一版已用 `MediaRecorder` + Pages Function 实现了一条较窄的省钱文字链路，但这不是对原始转写体系的完整迁移。
   - 后续应把原始代码中已经跑通过的设备接入、分段策略、转写合并策略抽成模块，而不是长期维护两套不一致实现。
   - 每段音频发送到服务端代理或 Pages Function。
   - 服务端调用 OpenAI 音频转写，再调用 Responses API 翻译为中文。
   - 页面只显示文字，不播放译声。

2. `实时译声模式`：
   - 保留现有 `gpt-realtime-translate` WebRTC 链路。
   - 作为需要最低延迟和中文译声时的可选模式。

## 测试策略

- 先写失败测试：
  - 页面默认模式为 `text` / `省钱文字模式`。
  - 页面包含控制抽屉和主阅读区。
  - 页面包含 60 分钟默认上限和倒计时提醒文案。
  - 文本模式模块能构造包含 base64 音频、语言和 mime type 的请求。
  - Cloudflare Function 能调用 transcription 和 Responses，并返回 `{ sourceText, targetText }`。
- 实现后跑全部现有测试和新增测试。

## 限制

- 蓝牙实体按键是否能被浏览器识别取决于设备和系统。第一版支持键盘事件，适合蓝牙遥控器或能映射成键盘输入的设备。
- 省钱文字模式会比实时译声模式有更高延迟，取决于分段时长，第一版默认 8 秒一段。
