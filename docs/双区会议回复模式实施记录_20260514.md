# 双区会议回复模式实施记录

日期：2026-05-14

## 目标

把实时翻译页面改成两个工作区：

- 我听别人：继续做实时听译，把会议/现场声音翻译成中文。
- 我要回复：用户说中文或输入中文，生成可照读的英文回复文本。

此方案不尝试直接控制 Zoom 静音，也不把 TTS 自动送进 Zoom 麦克风，避免会议中出现串音、回声和权限风险。

## 已实现

### 前端页面

入口仍为：

- `https://xcu.ai/trans/`

页面新增：

- `我听别人` 区：保留实时语音翻译、中文译文、播放器和延迟指标。
- `我要回复` 区：
  - 中文输入框
  - `说中文生成英文` 按钮，使用浏览器 `SpeechRecognition` / `webkitSpeechRecognition`
  - `生成英文回复` 按钮
  - `复制英文` 按钮
  - `清空` 按钮
  - 英文长句版
  - 英文短句版
  - 中文提示/发音提示
  - Zoom 静音提醒：念英文前后使用 `Alt + A`

如果浏览器不支持语音识别，页面会提示直接输入中文。

### Cloudflare Pages Function

新增接口：

- `/api/openai/reply-translation`
- `/api-proxy/api/openai/reply-translation`

接口使用 OpenAI Responses API：

- Endpoint：`https://api.openai.com/v1/responses`
- 默认模型：`gpt-5.2`
- 可通过 Cloudflare 环境变量 `OPENAI_REPLY_MODEL` 覆盖
- 输出结构：
  - `english`
  - `shortVersion`
  - `notes`

### Cloudflare Worker Router

`xcu-trans-router` 新增路由：

- `xcu.ai/api-proxy/api/openai/reply-translation`

这样 `xcu.ai/trans/` 页面可以继续使用相对路径调用回复接口。

### 本地 api-proxy fallback

本地 Express `api-proxy` 也新增：

- `/api/openai/reply-translation`
- `/api-proxy/api/openai/reply-translation`

用于本地开发或回退部署。

## 使用方式

1. 打开 `https://xcu.ai/trans/`
2. 左边 `我听别人` 用来听会议中文翻译。
3. 要回复时，先确认 Zoom 已静音。
4. 右边 `我要回复` 说中文或输入中文。
5. 点 `生成英文回复`。
6. 看英文，确认无误后按 `Alt + A` 取消 Zoom 静音。
7. 照着英文念。
8. 念完再按 `Alt + A` 静音。

## 限制

- 纯网页不能直接控制 Zoom 静音。
- 纯网页不能把英文 TTS 伪装成 Zoom 麦克风。
- 浏览器语音识别支持情况取决于浏览器，Chrome/Edge 通常更好。
- 在手机浏览器里切出 Zoom 可能影响页面录音或语音识别，因此正式会议建议用电脑。
